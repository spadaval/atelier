# Contributing To Atelier

Atelier uses its committed `.atelier/` tracker records as the source of truth
for planned work, active work, blockers, validation, and evidence. Do not open
or treat GitHub issues as the primary work queue for this repository.

Before changing code or docs:

1. Read [AGENTS.md](AGENTS.md).
2. Find or create the relevant Atelier issue.
3. Start tracked work with the workflow commands allowed by that issue.
4. Attach evidence that proves the issue outcome before closeout.

Pull requests should link the Atelier issue or mission they advance, summarize
the attached evidence, and include the validation commands that were run.
Dependency update pull requests are acceptable, but they still need normal
Atelier tracker ownership before merge.

Repository ownership and contribution-template decisions are recorded in
[Repository Contribution Policy](docs/product/repository-contribution.md).

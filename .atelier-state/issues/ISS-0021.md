Implement configurable workflows and gate-backed transitions from the repository-owned workflow configuration contract. Support record types, phases, transitions, required fields, done requirements, setup/validation hooks, and gates such as tests_passed, export_fresh, review_complete, evidence_attached, milestone_validated, and no_blocking_lints.

This should load explicit repo config rather than baking workflow policy into code or relying on hidden local setup.

## Acceptance Criteria

Workflow configuration is loaded from the chosen repo-owned contract and validated with stable errors; invalid transitions fail with actionable errors; gate checks produce stable machine-readable results; setup and validation hooks follow configured timeout/failure semantics; tests cover allowed transition, rejected transition, required evidence, invalid config, and config-driven lightweight versus strict workflows.

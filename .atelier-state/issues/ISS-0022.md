Add Braid/Worktrunk-style Git/worktree ergonomics while keeping Git as the underlying mechanism. Scope includes work start/finish, worktree creation, branch helpers, claim/work association, dirty-worktree protection, configured setup hooks, status surfaces, and rebuild in new worktrees.

Direct coding-agent process management, live agent-run tracking, retry queues, and session metrics are deferred from this milestone.

## Success Criteria

- Work start/finish and worktree helper flows exist with accepted command names.
- Work branches and worktrees are associated with records.
- Dirty-main and stale-export safeguards are enforced.
- New worktrees can rebuild SQLite state.
- Worktree status and setup behavior are driven by explicit workflow/config policy rather than hidden local conventions.

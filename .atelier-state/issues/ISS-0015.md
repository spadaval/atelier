Implement or rename the work-facing lifecycle commands around claims, branch/worktree association, and closeout gates, including `atelier work start <id>` and `atelier work finish <id>` or accepted command equivalents. The initial slice should not launch agents, supervise direct agent processes, or persist live run/session records.

Machine-local identity may remain a local configuration detail where needed for attribution, but durable run tracking is deferred until the workflow/evidence model is stable.

## Acceptance Criteria

Commands support stable JSON where automation needs it; work start records claim, branch, and worktree intent without creating live agent-run rows; work finish checks closure gates that exist at this stage; CLI integration tests cover happy path, dirty-worktree rejection, and no-agent-launch behavior.

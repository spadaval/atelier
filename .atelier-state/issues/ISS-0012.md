Expose enough mechanical state through JSON projections to support a future dashboard or local UI before building any rich interface.

Direct live agent-run tracking is deferred; the initial projection should focus on durable work state, workflow/config health, evidence, gates, branches, and worktrees.

## Success Criteria

- JSON projections cover active missions, milestone progress, blockers, workflow/config health, branches/worktrees, stale exports, evidence, gates, plan drift, recent decisions, and review/validation queues.
- Projection schema is documented and deterministic.
- Rich UI remains explicitly deferred until projections are useful.
- Live agent-run rows, retry queues, and session metrics are explicitly out of scope for the first projection milestone.

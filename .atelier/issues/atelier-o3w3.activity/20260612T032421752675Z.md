Start/work relationship decision.

Primary command: `atelier start <issue-id>` is the normal user-facing entrypoint. It associates the issue with current work, sets or verifies in-progress state, runs fast gate checks, and then applies configured workspace policy. Users should not need to understand sessions, branches, or worktrees to begin ordinary work.

Compatibility and old-to-new mapping:
- `atelier work start <issue-id>` -> compatibility alias for `atelier start <issue-id>` and retained for Agent Factory until guidance is migrated.
- `atelier issue start <issue-id>` -> do not add as a promoted public command because it expands the reduced issue surface. If it exists during migration, keep it hidden/deprecated and point to root `atelier start`.
- `atelier work finish <issue-id>` should map toward root `atelier finish <issue-id>` if a matching root finish command is introduced. Existing `work finish` remains a compatibility command until docs and Agent Factory switch.
- `atelier work status` remains useful as advanced/current-work inspection, but global/status signpost surfaces should expose the common answer.
- `atelier worktree for <issue-id>` remains advanced/orchestrator-facing for explicit worktree preparation, not required by default start.

Workspace policy model:
- `current-checkout`: default for prototypes and single-person repos; allows work in the current checkout, including main branch when policy allows it.
- `branch`: create or reuse a branch based on a template such as `codex/<issue-id>-<slug>`.
- `worktree`: create or locate a linked worktree using a path template and branch template.
- main branch behavior is separately configurable as allow, warn, or block. Default is allow or warn, not block, so first use is low friction.
- Per-command overrides may be flags such as `--workspace current|branch|worktree`, `--branch <name>`, `--path <path>`, and `--force-main` where policy permits.

Fast gate checks: start checks issue existence, closed/blocked state, projection freshness, basic lint readiness for the issue, active-work conflicts, and workspace policy feasibility. It must not run expensive validation or require evidence proof synchronously.

Help and guidance expectations: root help should list `atelier start <issue-id>` as the ordinary start path. `work start` help should say it is a compatibility/advanced form and point to root start. Agent Factory guidance can continue using `work start` until the root command exists and transcript tests cover default current-checkout, branch, worktree, and blocked/warned main-branch behavior.

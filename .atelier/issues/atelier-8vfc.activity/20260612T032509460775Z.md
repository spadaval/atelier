Help-text expectations for the command hierarchy.

Top-level help should preserve the group scan model: Setup/bootstrap, Issues, Missions/planning, Records/evidence, Work/start, State, Integrations, Maintenance, and Advanced/compatibility where needed. The common commands list should favor ordinary workflows: `atelier start <issue-id>`, `atelier issue create`, `atelier issue list --ready`, `atelier issue show`, `atelier mission status`, `atelier work status`, `atelier lint`, and `atelier doctor` or their preferred grouped equivalents once implemented.

Group help expectations:
- `atelier issue --help`: lifecycle essentials only in the primary list: create, list, show, update, close. Moved commands should either be absent, hidden, or clearly compatibility-only.
- `atelier issue create --help`: explain file-first creation, canonical Markdown path output, `--parent <id>`, `--work`, and lint/validation next step.
- `atelier start --help`: ordinary entrypoint for beginning work, with current-checkout default and branch/worktree policy options.
- `atelier work --help`: compatibility/current-work inspection and finish commands; point users to root start for normal beginning work.
- `atelier worktree --help`: advanced/orchestrator workspace management, not required for normal start.
- state help: distinguish export from rebuild and explain canonical Markdown versus rebuildable local projection.
- maintenance help: lint, doctor, diagnostics, destructive commands, and when to use each.
- `workflow validate --help`: advanced guardrail/policy validation, not the normal next-action discovery path.

Compatibility/deprecation notes: deprecated aliases should not crowd primary help. Command-specific help may include a short Migration or Compatibility line naming the preferred replacement.

Acceptance remains implementation-dependent: this note defines expected help behavior only. The issue should stay open until Rust help strings and transcript/help snapshots prove the expected text, boundaries, concise scanability, and compatibility notes.

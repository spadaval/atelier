Planning contract: root status, mission status, and mission show split.

`atelier status` is the first checkout-orientation command. It answers: where am I, is local state safe to act on, what mission/work is active, what work is ready now, what is blocked, and what should I run next. It should summarize mission identity and health but not print the mission body, full linked-work inventory, or full closeout validator transcript. It should include intent-labeled drill-down commands such as `atelier mission status <id>` for mission advancement, `atelier issue show <id>` for work detail, `atelier history --mission <id>` for activity, `atelier doctor` or `atelier export --check` for health failures, and `git status --short --branch` when the git state needs direct inspection.

`atelier mission status [<id>]` is the operational mission drill-down. With an ID it answers whether the mission can advance or close: ready linked work, blocked linked work, blockers by ID, evidence gaps, workflow validator failures, closeout gates, and next commands. Without an ID it should resolve to the one active mission when there is exactly one active mission; if there is none, print a clear empty state with mission list/create guidance; if more than one active mission can exist, print an ambiguity state with candidate IDs and require an ID. It should not replace `mission show` as the durable narrative view.

`atelier mission show <id>` is the durable record/document view. It prints identity, body, constraints, risks, validation expectations, linked records, and recent activity appropriate for understanding the mission record. It may summarize progress, but closeout readiness, workflow validator details, and next operational actions belong primarily in mission status.

Success criteria: transcripts prove the three commands do not produce overlapping half-document/half-status output, no-ID mission status respects the active-mission invariant, and every summarized status section has an obvious drill-down path. Regression tests should cover root checkout orientation, mission status with ID, no-ID mission status empty/active/ambiguous cases, and mission show remaining a document view.

Issue command surface decision.

Primary `atelier issue` help should be limited to lifecycle essentials:
- Keep: `issue create`, `issue list`, `issue show`, `issue update`, `issue close`.
- Fold into lifecycle commands: `issue quick` -> `issue create --work`; `issue subissue` -> `issue create --parent <id>`; `issue reopen` -> `issue update <id> --status open`; `issue label` -> `issue update <id> --label <label>`; `issue unlabel` -> `issue update <id> --remove-label <label>`; `issue blocked` -> `issue list --blocked`.
- Move to dependency/link/search/activity/hierarchy homes: `issue block` and `issue unblock` -> `atelier dep add/remove` or equivalent dependency group; `issue relate`, `issue unrelate`, `issue related` -> typed link group; `issue impact` and `issue tree` -> graph/hierarchy views; `issue search` -> cross-record `atelier search`; `issue comment` -> note/activity surface.
- Move to maintenance: `issue delete` -> maintenance/destructive command group with explicit confirmation expectations; `issue close-all` -> maintenance/bulk operation if retained at all.
- Remove or hide from normal help: `issue next` because ready/mission/status surfaces should own next-action guidance; `issue tested` because validation proof should be evidence-backed rather than a mutable issue flag.

Alias/deprecation policy: folded or moved commands may remain as hidden compatibility aliases for one staged migration when existing habits or Agent Factory guidance would break. They must print concise preferred-command guidance in normal output and stay out of primary help. Direct removal is acceptable for `issue next`, `issue tested`, and eventually `issue quick`/`issue subissue` only after replacement transcripts and docs prove the new commands cover the workflow.

Validation expectations: transcript tests should cover create, parented create, active-work create, reopen, label/unlabel, blocked listing, dependency add/remove/list, link add/remove/list, hierarchy/impact views, search, note/activity entry, and destructive deletion or bulk-close behavior where retained. `atelier issue --help` should scan as lifecycle-only, while advanced homes cover the moved commands.
